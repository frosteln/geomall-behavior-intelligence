Why Haiku wins
  • Best balance of quality, speed, and ops simplicity.
  • Fits OCR + extraction + normalization in one flow.
  • Strongest option for a fast production launch.
Why Hunyuan matters
  • Most credible self-host OCR path.
  • Better fit than generic VLMs for OCR-first work.
  • Useful if security or lock-in pushes against APIs.
Where others fit
  • Databricks: easiest pilot if your flow is already there.
  • Qwen: reusable multimodal platform, not the sharpest OCR tool.
  • Tesseract: benchmark baseline only.
Recommendation

Simple decision path for management

Phase 1

Ship with Claude Haiku 4.5

Use it to prove end-to-end OCR, extraction, Excel output, confidence scoring, and batch operations quickly.

Phase 2

Run Hunyuan OCR bake-off

Compare on the same Vietnamese banking sample set for field accuracy, exception rate, and all-in cost.

Keep in reserve

Use Databricks or Qwen only with a clear reason

Databricks for platform convenience. Qwen for broader multimodal reuse. Neither is the first OCR pick here.

“The best first answer is the option that gets reliable structured output into production fastest, with acceptable accuracy and the least operational drag.”
